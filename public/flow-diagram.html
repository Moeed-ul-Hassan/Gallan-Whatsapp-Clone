Gallan App Flow Diagram

Understanding the architecture and data flow

Application Overview

Authentication Flow

1
User Registration
New users register by providing a username, password, and optional profile details. The password is hashed before storage. 
POST /api/auth/register
Body: {
  "username": "user123",
  "password": "password123",
  "displayName": "User Name",
  "status": "Hey there! I'm using Gallan"
}
2
User Login
Users login with their username and password. The server verifies credentials and returns a session cookie. 
POST /api/auth/login
Body: {
  "username": "user123",
  "password": "password123"
}
3
Session Management
The application uses Express sessions with Passport.js for authentication. User sessions are maintained on the server.
When a user logs in, their online status is updated in the database, and contacts can see when they're active.
4
User Logout
Users can logout which destroys their session and updates their online status to offline. 
POST /api/auth/logout

Messaging Flow

1
Contact Management
Users can add contacts by username. The system verifies if the username exists and creates a contact relationship. 
POST /api/contacts
Body: {
  "username": "friend123",
  "displayName": "My Friend"
}
2
Chat Creation
When a user selects a contact to message, a chat is created if it doesn't already exist. Both users are added as participants. 
POST /api/chats
Body: {
  "contactId": 2,
  "name": "Chat with Friend"
}
3
Sending Messages
Users can send text messages within a chat. The message is stored with the sender's ID and a timestamp. 
POST /api/messages
Body: {
  "chatId": 1,
  "text": "Hello, how are you?"
}
4
Message Delivery & Status
When a message is sent, a message status entry is created for each participant (except the sender). Status can be 'delivered' or 'read'.
The application uses polling to check for new messages periodically. In a production environment, WebSockets would provide real-time updates.

API Endpoints

Authentication

POST /api/auth/register
Register a new user account
POST /api/auth/login
Authenticate a user and create a session
GET /api/auth/me
Retrieve current authenticated user's data
POST /api/auth/logout
End the current user session

Users

PATCH /api/users/:id
Update user profile information

Contacts

GET /api/contacts
Get a list of user's contacts
POST /api/contacts
Add a new contact

Chats

GET /api/chats
Get a list of user's chats
POST /api/chats
Create a new chat or return existing chat

Messages

GET /api/messages/:chatId
Get messages for a specific chat
POST /api/messages
Send a message in a chat
